# Súgó v1.1 — Options Menu, Model Selector & Graceful Shutdown

## Overview

Add an options menu to the Súgó UI with a Whisper model size selector and a quit button for graceful shutdown. These are power-user features that stay out of the way of the primary transcription flow.

## Feature 1: Options Menu + Model Selector

### UI

- **Gear icon (⚙️)** in the header, left side (opposite the HU/EN toggle)
- Clicking the gear opens a **dropdown panel** that overlays the page content
- Styled to match the dark theme: `#1a1a1a` background, `#2a2a2a` border, rounded corners, subtle shadow
- Panel closes when clicking outside of it
- The options panel manages its own open/closed state independently of the main `setState()` function — it overlays the existing UI states rather than replacing them

### Model Selector

Inside the options panel, a labeled dropdown:

- Label: "Modell méret" / "Model size" (i18n)
- Options:
  - **"Auto (ajánlott)" / "Auto (recommended)"** — current behavior, selects based on system RAM (medium for ≤8GB, large-v3 for 16GB+). This is the default.
  - **"Medium — gyorsabb" / "Medium — faster"** — forces medium model (~1.5GB RAM)
  - **"Large-v3 — pontosabb" / "Large-v3 — more accurate"** — forces large-v3 model (~3GB RAM)
- Selection is persisted to `localStorage` under key `sugo_model_size` (underscore, consistent with existing `sugo_lang` key)
- On page load, the saved preference is restored

### Backend Changes

- **Modify `POST /api/load-model`**: Accept an optional JSON body `{"model_size": "medium" | "large-v3" | null}` via a Pydantic model with all-optional fields. When no body is sent (existing page-load call) or `model_size` is `null`, use auto-detection (current behavior). When specified, force that model size. The existing no-body call must continue to work — use `request.body()` to detect empty body and fall back to defaults.
- **Modify `transcribe.py`**: Add a `reload_model(model_size)` function that calls `load_model.cache_clear()` (Python's `lru_cache` only supports clearing all entries, not per-key) and then calls `load_model(model_size)` to warm the new model.
- **Modify `POST /api/transcribe`**: Accept an optional `model_size` form field. Pass it through to `transcribe_audio()`. This is necessary because without it, the transcription route falls back to auto-detected size rather than the user's selection.
- **Modify `GET /api/model-status`**: Return the currently active model size (from the loaded model), not the auto-detected size, so the frontend reflects what's actually in use.

### Model Switch Flow

1. User selects a different model size in the options panel
2. Frontend saves to `localStorage`
3. Frontend calls `POST /api/load-model` with the new model size
4. Backend clears the old cached model and loads the new one
5. UI shows a brief loading state during the switch
6. Next transcription uses the new model

## Feature 2: Graceful Shutdown

### UI

- **"Kilépés" / "Quit" button** inside the options panel, below the model selector
- Separated from settings by a thin divider line
- Styled as a subtle text button with red-tinted color (`#ef4444` or similar) — distinct but not alarming
- Clicking it:
  1. Shows "Súgó leáll..." / "Súgó shutting down..." in the main content area
  2. Calls `POST /api/shutdown`
  3. After response (or timeout), shows: "Bezárhatod ezt az ablakot." / "You can close this window."

### Backend

- **New `POST /api/shutdown` endpoint**:
  - Returns `{"status": "shutting_down"}` immediately
  - After a 0.5s delay (via `threading.Timer`), calls `os._exit(0)` to terminate the process
  - The delay ensures the HTTP response is sent before the process exits
  - Note: `os._exit(0)` is a hard kill that bypasses `atexit` handlers and `finally` blocks. This is acceptable for a local tool — any in-flight transcription is abandoned. The quit button should be disabled during active transcription to prevent accidental data loss.

### Launcher Update

- After the `uv run python -m transcriber.app` command exits, `start.command` prints:
  ```
  Súgó stopped. You can close this window.
  ```
- Note: `start.command` uses `set -e`, so this message only prints on clean exits (exit code 0, i.e. quit button). Ctrl+C (exit code 130) will cause the shell to exit before printing. This is acceptable — the Terminal window itself closing is sufficient signal.

## Non-Goals

- Speaker diarization (deferred — evaluating Senko library first)
- Auto-shutdown on idle timeout
- Settings persistence beyond localStorage (no config files)
